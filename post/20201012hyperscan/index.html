Hyperscan是一款来自于Intel的高性能的正则表达式匹配库。它是基于X86平台以PCRE为原型而开发的。在支持PCRE的大部分语法的前提下，Hyperscan增加了特定的语法和工作模式来保证其在真实网络场景下的实用性。

1. 概述

Hyperscan demo中使用libpcap从pcap文件中读取数据包，并根据一个规则文件中指定的多个正则表达式对报文进行匹配，并输出匹配结果和一些统计信息。pcapscan使用并对比了两种匹配模式：BLOCK和STREAM。BLOCK模式时它对单个数据包进行匹配；而STREAM模式下它通过五元组将数据包进行简单分流，并对每条流中的数据进行匹配。STREAM模式可以命中跨越数据包边界的匹配数据（比如，要匹配abc，而a在前一个数据的末尾，而bc在后一个数据包的前端，这两个数据包在一个流中，那么STREAM模式匹配可以命中它，而BLOCK模式不能）。

2. Hyperscan源码

2.1 编译

函数buildDatabase用来编译规则文件中的多个正则表达式，参数mode指定了是BLOCK还是STREAM模式。

 1static hs_database_t *buildDatabase(const vector<const char *> &expressions,
 2                                    const vector<unsigned> flags,
 3                                    const vector<unsigned> ids,
 4                                    unsigned int mode) {
 5    hs_database_t *db;
 6    hs_compile_error_t *compileErr;
 7    hs_error_t err;
 8
 9    Clock clock;
10    clock.start();
11
12    err = hs_compile_multi(expressions.data(), flags.data(), ids.data(),
13                           expressions.size(), mode, nullptr, &db, &compileErr);
14
15    clock.stop();
16
17    if (err != HS_SUCCESS) {
18        if (compileErr->expression < 0) {
19            // The error does not refer to a particular expression.
20            cerr << "ERROR: " << compileErr->message << endl;
21        } else {
22            cerr << "ERROR: Pattern '" << expressions[compileErr->expression]
23                 << "' failed compilation with error: " << compileErr->message
24                 << endl;
25        }
26        // As the compileErr pointer points to dynamically allocated memory, if
27        // we get an error, we must be sure to release it. This is not
28        // necessary when no error is detected.
29        hs_free_compile_error(compileErr);
30        exit(-1);
31    }
32//...
33}

其中的核心代码是hs_compile_multi的调用，此函数用来编译多个正则表达式，从代码可见除了mode参数，BLOCK和STREAM模式都使用这一API。它的原型是

1hs_error_t hs_compile_multi(const char *const * expressions, 
2                            const unsigned int * flags, 
3                            const unsigned int * ids, 
4                            unsigned int elements, 
5                            unsigned int mode, 
6                            const hs_platform_info_t * platform, 
7                            hs_database_t ** db, 
8                            hs_compile_error_t ** error)

其中，expressions是多个正则表达式字符串，flags和ids分别是expressions对应的flag和id数组；elements是表达式字符串的个数；其余参数与上一个例子中提到的hs_compile的参数涵义相同。

这里要注意的一个事情是参数ids，它是正则表达式的ID数组。每个表达式都有一个唯一ID，这样命中的时候匹配回调函数可以得到此ID，告诉调用者哪个表达式命中了。如果ids传入NULL，则所有表达式的ID都为0。

2.2 准备匹配临时数据

Benchmark构造函数中，为接下来的匹配分配足够的临时数据空间(scratch space）。这里有一个技巧：1）BLOCK和STREAM模式的匹配只需共用一个scratch；2）这个scratch足够大，方法是调用两次，在第2次调用时hyperscan如果发现空间不够会进行增加。

 1public:
 2    Benchmark(const hs_database_t *streaming, const hs_database_t *block)
 3        : db_streaming(streaming), db_block(block), scratch(nullptr),
 4          matchCount(0) {
 5        // Allocate enough scratch space to handle either streaming or block
 6        // mode, so we only need the one scratch region.
 7        hs_error_t err = hs_alloc_scratch(db_streaming, &scratch);
 8        if (err != HS_SUCCESS) {
 9            cerr << "ERROR: could not allocate scratch space. Exiting." << endl;
10            exit(-1);
11        }
12        // This second call will increase the scratch size if more is required
13        // for block mode.
14        err = hs_alloc_scratch(db_block, &scratch);
15        if (err != HS_SUCCESS) {
16            cerr << "ERROR: could not allocate scratch space. Exiting." << endl;
17            exit(-1);
18        }
19    }

2.3 读取数据包、分流

在Benchmark::readStreams方法中，从pcap文件中读取了所有数据包（其实封装必须是ethernet-ipv4-tcp/udp），并根据五元组进行简单分流。主要代码如下

 1while ((pktData = pcap_next(pcapHandle, &pktHeader)) != nullptr) {
 2            unsigned int offset = 0, length = 0;
 3            if (!payloadOffset(pktData, &offset, &length)) {
 4                continue;
 5            }
 6
 7            // Valid TCP or UDP packet
 8            const struct ip *iphdr = (const struct ip *)(pktData
 9                    + sizeof(struct ether_header));
10            const char *payload = (const char *)pktData + offset;
11
12            size_t id = stream_map.insert(std::make_pair(FiveTuple(iphdr),
13                                          stream_map.size())).first->second;
14
15            packets.push_back(string(payload, length));
16            stream_ids.push_back(id);
17        }

注意，stream_ids这个vector存储了每一个数据包对应的stream id。

2.4 打开流

由于需要用到STREAM模式，所以在匹配前要先将流打开，见Benchmark::openStreams

 1// Open a Hyperscan stream for each stream in stream_ids
 2    void openStreams() {
 3        streams.resize(stream_map.size());
 4        for (auto &stream : streams) {
 5            hs_error_t err = hs_open_stream(db_streaming, 0, &stream);
 6            if (err != HS_SUCCESS) {
 7                cerr << "ERROR: Unable to open stream. Exiting." << endl;
 8                exit(-1);
 9            }
10        }
11    }

其中，streams的类型是vector<hs_stream_t *>。

2.5 匹配

2.5.1 STREAM模式

在Benchmark::scanStreams中

 1// Scan each packet (in the ordering given in the PCAP file) through
 2    // Hyperscan using the streaming interface.
 3    void scanStreams() {
 4        for (size_t i = 0; i != packets.size(); ++i) {
 5            const std::string &pkt = packets[i];
 6            hs_error_t err = hs_scan_stream(streams[stream_ids[i]],
 7                                            pkt.c_str(), pkt.length(), 0,
 8                                            scratch, onMatch, &matchCount);
 9            if (err != HS_SUCCESS) {
10                cerr << "ERROR: Unable to scan packet. Exiting." << endl;
11                exit(-1);
12            }
13        }
14    }

hs_scan_stream的原型：

1hs_error_t hs_scan_stream(hs_stream_t * id, 
2                          const char * data, 
3                          unsigned int length, 
4                          unsigned int flags, 
5                          hs_scratch_t * scratch, 
6                          match_event_handler onEvent, 
7                          void * ctxt)

其中，id是数据所属的stream对应hs_stream_t指针，这里叫id其实我感觉不太合适; 其余参数与hs_scan相同。 这里调用的streams[stream_ids[i]]已经在上一步打开流中初始化。

2.5.2 BLOCK模式

BLOCK模式比STREAM简单许多，在Benchmark::scanBlock中

 1// Scan each packet (in the ordering given in the PCAP file) through
 2    // Hyperscan using the block-mode interface.
 3    void scanBlock() {
 4        for (size_t i = 0; i != packets.size(); ++i) {
 5            const std::string &pkt = packets[i];
 6            hs_error_t err = hs_scan(db_block, pkt.c_str(), pkt.length(), 0,
 7                                     scratch, onMatch, &matchCount);
 8            if (err != HS_SUCCESS) {
 9                cerr << "ERROR: Unable to scan packet. Exiting." << endl;
10                exit(-1);
11            }
12        }
13    }

其中，db就是上一步编译的databas；data和length分别是要匹配的数据和数据长度；flags用来在未来版本中控制函数行为，目前未使用；scratch是匹配时要用的临时数据，之前已经分配好；onEvent非常关键，即匹配时调用的回调函数，由用户指定；context是用户自定义指针。

2.5.3 匹配回调函数

匹配回调函数的原型是

1typedef (* match_event_handler)(unsigned int id, 
2                                unsigned long long from, 
3                                unsigned long long to, 
4                                unsigned int flags, 
5                                void *context)

其中，id是命中的正则表达式的ID，对于使用hs_compile编译的唯一表达式来说，此值为0；如果在编译时指定了相关模式选项(hs_compile中的mode参数），则此值将会设为匹配特征的起始位置，否则会设为0；to是命中数据的下一个字节的偏移；flags目前未用；context是用户自定义指针。

返回值为非0表示停止匹配，否则继续；在匹配的过程中，每次命中时都将同步调用匹配回调函数，直到匹配结束。

2.6 清理资源

包括关闭流（hs_close_stream）、释放database等。这里要注意hs_close_stream时仍会进行匹配。

3. STREAM模式总结

STREAM模式的用法比BLOCK模式要复杂一些，这里简单用伪代码总结一下

 1// N是流的规格，事先已确定好
 2hs_database_t* db;
 3hs_stream_t*  steams[N];
 4hs_scratch_t* tmp;
 5uint8_t* pkt;
 6
 7// 1) 编译多个正则表达式
 8hs_compile_multi(&db, HS_MODE_STREAM);
 9// 2) 准备scratch
10hs_alloc_scratch(db, &tmp);
11// 3) 打开流
12for(i=0; i<N; i++)
13  hs_open_stream(db, &streams[i]);
14// 4) 收到数据包，并将其分到指定流
15stream_id = classify(pkt);
16// 5) 流匹配
17hs_scan_stream(streams[stream_id], pkt, &tmp, callBack);
18// 6) 清理资源, 注意hs_close_stream仍可能有匹配
19for(i=0; i<N; i++)
20  hs_close_stream(db, streams[i], &tmp, callBack);hs_free_scrach(tmp);
21hs_free_database(db);

可以通过hs_database_size()和hs_stream_size()分别获得database和每条流的stream state的大小。正则表达式的数目和复杂度会影响stream state的大小，随着数目和复杂度的增加，可能会越来越大。在支持上百万条流和复杂规则文件的系统上，stream state的内存耗费可能很大。

4. 编译运行

运行示例前要准备一个pcap文件和一个规则文件，规则文件的格式如

1123：/weibo/i
2456：/[f|F]ile/Hi

id:pattern:flags

  • id: 可以随意定义，唯一即可
  • pattern： 正则，十六进制需加如“\x..\x..”
  • flags: 不同标记有不同匹配方式如：忽略大小写，逻辑匹配（将id与或非连接）
